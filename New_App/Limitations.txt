2. STEP-BY-STEP WIZARD vs. GROUPED FORMS
   - The new design appears to use a one-question-per-page wizard pattern (as seen in the
     marital status screenshot) rather than grouped sections.
   - Only the marital status sub-step was implemented as its own page because it was the only
     additional step visible in the examined screenshots.
   - If the remaining screenshots show additional individual question pages (e.g., employment
     status, occupation, homeowner status, etc.), those sub-steps are NOT included in this
     implementation and would need to be added.

3. QUOTE RESULTS PAGE (your-cover.png)
   - The your-cover.png image (final results page design) was NOT viewable due to the 5-image
     limit. The quote.html page was implemented based on the legacy app's quote logic and
     general Swinton Go! brand styling, but may not exactly match the intended design.

4. STUBBED API CALLS
   - The DVLA vehicle registration lookup is stubbed: entering "D500 NLE" returns a
     2019 Volkswagen Golf 1.5 TSI SE. Any other registration returns the same fallback
     vehicle (VW Golf) after a simulated 1.1-second delay.
   - The quote price calculation uses a deterministic algorithm (no real insurer API).
   - No real payment processing is implemented.

5. FONTS AND EXTERNAL ASSETS
   - The stylesheet loads Inter font from Google Fonts. In offline environments this will
     fall back to system sans-serif fonts.

6. BROWSER COMPATIBILITY
   - Tested with Playwright (Chromium). The app is built with standard HTML5/CSS3/ES6+
     and should work in all modern browsers. IE11 is not supported.

7. MISSING LEGACY CODE
   - The Legacy_Code/Legacy_BOS.zip contained a .NET (ASP.NET Core) project skeleton
     with mostly empty folders (no Views, Controllers, or Services with actual logic).
   - The Example_Website folder was used as the functional reference for the quote journey.

8. ACCESSIBILITY
   - Basic ARIA labels and semantic HTML are in place. A full WCAG 2.1 AA audit has not
     been conducted.
